Simple OpenCL Bitonic Sort.

  • Sorting values by keys
  • Fast; 8ms for 1M numbers, 207ms for 16M numbers on HD7970 (or may be 7950, even don't know exactly)
  • Fast; 13ms for 1M numbers, 307ms for 16M numbers on GTX670
  • Easy to integrate in existing opencl code
  • Pure OpenCL API
  • NO CMAKE
  • NO BOOST
  • NO cl.hpp
  • NO other stupid frameworks
  • NO additional memory allocations
  • Any key type is possible, easy to change
  • O(Nlog(N)log(N))
  • JUST WORKING AND FAST SORT, JUST TAKE AND USE!
